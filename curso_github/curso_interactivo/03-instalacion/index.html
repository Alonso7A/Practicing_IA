Capítulo 03

Instalación y configuración inicial

Prepara tu computadora: crea la cuenta de GitHub, instala Git y Visual Studio Code, y configura tu identidad para que los commits queden a tu nombre.

Para trabajar con GitHub necesitas tres cosas: una cuenta en la web, Git instalado en tu PC y un editor de código (idealmente Visual Studio Code).

Paso 1 — Crear cuenta en GitHub

Ve a github.com y pulsa Sign up. Vas a necesitar correo, usuario y contraseña. Confirma tu correo desde el enlace que te envíen.

Paso 2 — Instalar Visual Studio Code

Descárgalo desde code.visualstudio.com. Es el editor más usado y tiene integración nativa con Git, terminal incorporada y miles de extensiones.

Paso 3 — Instalar Git

Descárgalo desde git-scm.com/downloads. En Windows el instalador te hace varias preguntas. La siguiente tabla resume qué elegir si eres principiante:

Opciones recomendadas en la instalación de Git para Windows
PantallaOpción recomendadaPor qué
Editor por defectoVisual Studio CodeVim (el default) tiene una curva de aprendizaje frustrante. VS Code es intuitivo.
Nombre de rama inicialLet Git decideUsará master. Es válido. Hoy se prefiere main pero no es obligatorio.
Ajuste del PATHGit from the command line and 3rd-party softwarePermite usar Git desde CMD, PowerShell y la terminal de VS Code.
Cliente SSHUse bundled OpenSSHTrae su propia versión, compatible con todo.
Finales de líneaCheckout Windows-style, commit Unix-styleEvita problemas al colaborar con usuarios de Mac/Linux.

Paso 4 — Verificar la instalación

Abre la terminal en VS Code (Ctrl+`) y escribe:

git --version

Si te muestra algo como git version 2.43.0, la instalación funcionó.

Paso 5 — Configurar tu identidad en Git

Git necesita saber quién eres para "firmar" cada commit. Esto se hace una sola vez por computadora:

git config --global user.name "Tu Nombre"
git config --global user.email "tucorreo@ejemplo.com"
git config --list

Importante

El correo que pongas en user.email debería ser el mismo de tu cuenta de GitHub. Si no, los commits aparecerán como "no verificados" o atribuidos a una cuenta fantasma.

Paso 6 — (Opcional) Que los repos nuevos usen "main"

Por defecto, Git nombra la rama inicial como master. Hoy se prefiere main:

git config --global init.defaultBranch main
