Capítulo 07

GitHub desde Visual Studio Code

El panel Source Control, atajos clave, extensiones imprescindibles, cómo clonar repos y conectar tu cuenta sin escribir contraseñas cada vez.

VS Code tiene integración nativa con Git. No necesitas usar la terminal — todo se puede hacer con clics — pero combinar ambos te hace más rápido y entiendes lo que pasa por debajo.

El panel "Source Control"

Se abre con Ctrl+Shift+G o pulsando el icono con forma de bifurcación en la barra lateral izquierda. Verás:

  • Changes: archivos modificados que aún no están en staging.
  • Staged Changes: archivos listos para commitear.
  • Caja de mensaje: donde escribes el mensaje del commit.
  • Botón ✓ Commit: equivalente a git commit.

Clonar un repo desde VS Code (forma fácil de empezar)

  1. Pulsa Ctrl+Shift+P para abrir la paleta de comandos.
  2. Escribe Git: Clone y selecciónalo.
  3. Pega la URL del repo (que copias del botón Code en GitHub).
  4. Elige una carpeta de tu PC donde guardarlo.
  5. VS Code te pregunta si quieres abrir el repo → Open.

Inicializar un repo nuevo desde VS Code

Si tienes una carpeta con archivos y la quieres convertir en repo Git:

  1. Abre la carpeta en VS Code (File → Open Folder).
  2. Ve a Source Control (Ctrl+Shift+G).
  3. Pulsa Initialize Repository.
  4. Verás todos tus archivos como U (untracked).

Conectar VS Code con tu cuenta de GitHub

  1. Pulsa Ctrl+Shift+P y escribe Sign in to GitHub.
  2. Se abre el navegador → Authorize.
  3. Vuelves a VS Code. Listo: ahora puedes hacer push/pull sin que te pida contraseña.

Extensiones imprescindibles

  • GitHub Pull Requests and Issues — gestiona PRs sin salir de VS Code.
  • GitLens — muestra quién escribió cada línea (blame) y la historia de cada cambio.
  • Git Graph — visualización del historial como árbol.
  • Git History — explora commits y diferencias.

Tip

Cuando pulsas Commit sin nada en staging, VS Code te pregunta si quieres hacer stage all. Si dices que sí, equivale a git add . seguido de git commit. Útil pero peligroso si tienes archivos que no querías incluir — siempre revisa el panel antes.
