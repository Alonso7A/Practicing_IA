Ramas (branches): el corazón de Git

Una línea paralela de desarrollo para probar ideas sin romper lo que ya funciona. Aprende a crear, cambiar, fusionar y borrar ramas con seguridad.

Una rama es una línea paralela de desarrollo. Te permite probar ideas, desarrollar funciones nuevas o arreglar bugs sin tocar el código que ya funciona.

Imagina esto

Tu rama principal (main) es como el manuscrito final de un libro. Cuando quieres agregar un capítulo nuevo, no escribes directamente sobre el original: sacas una copia (rama), escribes ahí, lo revisas, y solo cuando estás seguro, lo agregas al manuscrito final (merge).

Crear y trabajar con ramas

# Ver en qué rama estás y qué ramas existen
git branch

# Crear y cambiar a una rama nueva
git checkout -b feature/login

# Hacer cambios, agregar, commit...
git add .
git commit -m "implementa formulario de login"

# Subir la rama nueva al remoto
git push -u origin feature/login

# Volver a la rama principal
git checkout main

# Fusionar la rama de feature en main
git merge feature/login

# Borrar la rama ya fusionada (opcional)
git branch -d feature/login

Regla de oro

Nunca trabajes directamente en main en proyectos serios. Siempre crea una rama, haz tus cambios ahí, y luego pide que se fusione vía Pull Request. Esto protege la rama principal de errores y permite revisión.

En VS Code

Abajo a la izquierda ves el nombre de la rama actual (ej: main). Clic ahí → puedes:

  • Cambiar de rama.
  • Crear una nueva.
  • Ver todas las que existen.
