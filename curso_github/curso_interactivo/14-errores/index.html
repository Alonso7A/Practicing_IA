Capítulo 14

Errores comunes y cómo resolverlos

Los siete tropiezos que más asustan a los principiantes y la forma exacta de salir de ellos. Que no cunda el pánico.

1. "fatal: not a git repository"

Estás en una carpeta donde no hay git init. Solución:

git init
# O verifica que estás en la carpeta correcta:
pwd

2. "Updates were rejected because the remote contains work..."

El remoto tiene commits que tu local no tiene. Solución:

git pull origin main
# Si avisa de "histories no relacionadas":
git pull origin main --allow-unrelated-histories
# Luego:
git push

3. "Please tell me who you are"

No has configurado tu identidad. Solución:

git config --global user.name "Tu Nombre"
git config --global user.email "tucorreo@ejemplo.com"

4. "Authentication failed" al hacer push

GitHub no acepta contraseña. Genera un PAT (ver Capítulo 12 — Seguridad) o configura SSH.

5. Subiste un archivo con contraseñas por error

Acción urgente

1. Considera la contraseña/key comprometida: cámbiala YA en el servicio correspondiente. Aunque borres el commit, ya quedó expuesta.

2. Para borrar el archivo del historial: git filter-branch o la herramienta BFG Repo-Cleaner.

3. Agrega el archivo a .gitignore para que no vuelva a pasar.

6. Conflictos de merge

Dos personas modificaron las mismas líneas. Git no sabe qué versión usar. Cuando abres el archivo verás algo así:

<<<<<<< HEAD
const color = "azul";
=======
const color = "rojo";
>>>>>>> feature/dark-mode

Edita el archivo dejando solo la versión correcta (quita los marcadores <<<, ===, >>>), guarda, y haz:

git add archivo-en-conflicto.js
git commit -m "resuelve conflicto de merge en color principal"

VS Code te ayuda con conflictos

Cuando abres un archivo en conflicto, VS Code muestra botones "Accept Current Change", "Accept Incoming Change", "Accept Both". Úsalos en lugar de editar manualmente.

7. Olvidaste hacer pull y ahora tienes conflictos

Si tus cambios son pequeños y no quieres lidiar con conflictos:

# Guarda tus cambios en el cajón
git stash

# Baja los últimos cambios del remoto
git pull

# Recupera tus cambios encima de los del remoto
git stash pop

# Si hay conflictos ahora, resuélvelos manualmente
