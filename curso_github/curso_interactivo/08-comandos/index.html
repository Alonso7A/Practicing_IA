Capítulo 08

Comandos esenciales de Git

La referencia completa, organizada por categoría. No los memorices todos: usa los básicos a diario y consulta los demás cuando los necesites.

Configuración inicial

ComandoFunción
git config --global user.name "nombre"Configura tu nombre de usuario
git config --global user.email "email"Configura tu correo
git config --listMuestra la configuración actual
git config --global init.defaultBranch mainHace que los repos nuevos usen main en lugar de master

Comandos básicos del día a día

ComandoFunción
git initInicializa un repositorio Git en la carpeta actual
git clone [url]Descarga un repo remoto completo
git statusEstado actual: archivos modificados, en staging, sin trackear
git add [archivo]Agrega un archivo al staging
git add .Agrega todos los archivos modificados
git commit -m "mensaje"Crea un commit con los archivos en staging
git logHistorial completo de commits
git log --oneline --graphHistorial visual compacto, mostrando ramas

Sincronización con GitHub

ComandoFunción
git remote add origin [url]Conecta el repo local con uno remoto
git remote -vMuestra los remotos configurados
git push origin [rama]Sube los commits al remoto
git pull origin [rama]Descarga y fusiona los cambios del remoto
git fetchDescarga cambios sin fusionarlos (más seguro que pull)

Ramas y fusión

ComandoFunción
git branchLista las ramas locales (marca la activa con *)
git branch [nombre]Crea una nueva rama
git checkout [rama]Cambia a otra rama
git checkout -b [rama]Crea y cambia a una nueva rama (atajo)
git switch [rama]Versión moderna de checkout (más segura)
git merge [rama]Fusiona otra rama dentro de la actual
git branch -d [rama]Elimina una rama (si ya fue fusionada)

Avanzados y de rescate

ComandoFunción
git diffMuestra qué cambió línea por línea
git reset [archivo]Quita un archivo del staging (sin perder cambios)
git restore [archivo]Descarta los cambios en un archivo modificado
git stashGuarda cambios temporalmente (cajón)
git stash popRecupera los cambios guardados con stash
git tag [nombre]Etiqueta un commit (típicamente versiones: v1.0)
git show [commit]Detalles de un commit específico
git blame [archivo]Quién modificó cada línea y cuándo
git revert [commit]Crea un commit que deshace otro (seguro, no borra historial)

Los 6 comandos del 80% del trabajo

Si tuvieras que recordar solo seis: git status, git add, git commit, git push, git pull y git checkout -b. Con esos manejas el 80% de tu trabajo diario.
