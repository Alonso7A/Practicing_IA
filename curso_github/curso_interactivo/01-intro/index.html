Capítulo 01

¿Qué es Git y qué es GitHub?

Entiende de una vez por todas la diferencia entre la herramienta que vive en tu computadora y la plataforma que vive en la nube.

Dos cosas distintas que se mencionan juntas

Aunque suelen aparecer en la misma frase, Git y GitHub no son lo mismo. Entender la diferencia es el primer paso para no confundir conceptos.

Definición

Git es un sistema de control de versiones distribuido: un programa que se instala en tu computadora y registra cada cambio que haces en los archivos de un proyecto, permitiéndote volver atrás, comparar versiones y trabajar en paralelo con otras personas.

GitHub es una plataforma en la nube que aloja repositorios Git, añade colaboración (issues, pull requests, code review), respaldo, hosting de páginas, automatizaciones (Actions) y se ha convertido en el portafolio profesional estándar.

Analogía sencilla

Git es el cuaderno donde anotas todos los cambios, quién los hizo y cuándo.

GitHub es la biblioteca en internet donde guardas ese cuaderno, con mesas para reunirse, ventanas para que otros vean tu trabajo y estantes para organizar los capítulos.

El "porqué" de todo esto

Imagina que estás escribiendo un libro en Word. Cada vez que haces un cambio importante, guardas una copia: libro_v1.docx, libro_v2.docx, libro_v3_final.docx, libro_v3_final_DEFINITIVO.docx... 😅

Git hace eso automáticamente, ordenado, sin duplicar archivos pesados, y te deja volver a cualquier versión anterior cuando quieras. Y GitHub guarda esas versiones en internet para que no las pierdas y puedas compartirlas.

¿Para qué se usa todo esto en la vida real?

  • Llevar control de cambios en proyectos personales o de equipo.
  • Volver atrás cuando algo se rompe sin tener que adivinar qué cambió.
  • Colaborar con otras personas sin pisarse el trabajo entre sí.
  • Respaldar tu código en la nube por si tu computadora se daña.
  • Mostrar tu portafolio profesional a empresas que quieran contratarte.
  • Publicar páginas web gratis con GitHub Pages.
