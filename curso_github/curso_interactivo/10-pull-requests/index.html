Pull Requests: la magia colaborativa

La pieza central de la colaboración en GitHub. Aprende a proponer cambios, recibir revisión, contribuir a proyectos ajenos con forks y hacer merges responsables.

Un Pull Request (o Merge Request en GitLab) es una solicitud formal para que tu rama se fusione con otra (típicamente main). Antes de hacerlo, otras personas pueden revisar tu código, dejar comentarios, sugerir cambios y aprobarlo.

Crear un Pull Request paso a paso

  1. Haz tus cambios en una rama (nunca directamente en main).
  2. Sube la rama: git push -u origin feature/x
  3. Ve a GitHub. Aparecerá un banner amarillo: "Compare & pull request". Púlsalo.
  4. Escribe un título claro y una descripción que explique qué y por qué.
  5. Asigna reviewers si trabajas en equipo.
  6. Pulsa Create pull request.
  7. Espera la revisión. Si piden cambios, hazlos en tu rama local y haz push: el PR se actualiza solo.
  8. Cuando aprueben, pulsa Merge pull request y luego Delete branch.

Hacer un Fork (contribuir a proyectos ajenos)

  1. En el repo ajeno, pulsa Fork (arriba a la derecha). Eso crea una copia en tu cuenta.
  2. Clonas tu copia: git clone https://github.com/tu-usuario/repo.git
  3. Creas una rama, haces cambios, los subes a tu fork.
  4. En GitHub, abres un Pull Request desde tu fork hacia el repo original.
  5. El dueño del proyecto revisa y decide si aceptar tus cambios.

El secreto de un buen PR

Los estudios muestran que la calidad de la revisión cae bruscamente en PRs de más de 400 líneas — los revisores dejan de leer con atención. PRs pequeños = mejor código. Cada PR debe representar un único cambio lógico.

Cómo escribir una buena descripción de PR

## Qué cambia
Breve explicación de qué hace este PR.

## Por qué
Motivación: bug que arregla, feature que añade, problema que resuelve.

## Cómo probarlo
Pasos para que el revisor verifique que funciona.

## Capturas (si aplica)
Imágenes antes / después si es un cambio visual.

Resumen mental

PR = "Hola, hice estos cambios en mi rama, ¿pueden revisarlos antes de fusionar a main?". Es la conversación oficial sobre el código.
