Crear un repositorio desde la web de GitHub

Aprende a crear repos, subir archivos arrastrándolos, editar desde el navegador y conectarlo todo a tu carpeta local. Sin tocar la terminal si no quieres.

Hay dos caminos para empezar un proyecto: crear el repo en GitHub primero y luego clonarlo, o crearlo localmente y luego conectarlo. Vamos a empezar por el camino más limpio: GitHub primero.

Pasos para crear un repo en GitHub

  1. PASO 1: Inicia sesión en github.com. En la esquina superior izquierda pulsa el icono de las tres rayas y selecciona Repositories.
  2. PASO 2: Pulsa New repository. Escribe un Repository name (sin espacios, usa guiones: mi-proyecto) y una Description breve.
  3. PASO 3: Elige Public o Private. Público lo ve todo el mundo; privado solo tú e invitados. Para empezar, público es buena opción (GitHub Pages necesita repos públicos en el plan gratuito).
  4. PASO 4: Marca Add a README file para que el repo no esté vacío. Opcionalmente agrega .gitignore según el lenguaje, y elige una License si vas a compartir (MIT es la más común).
  5. PASO 5: Pulsa Create repository. ¡Listo! GitHub te muestra la URL del repo: cópiala.

Subir archivos arrastrándolos (sin terminal)

La forma más fácil sin tocar comandos:

  1. Entra a tu repo en GitHub.
  2. Botón Add file → Upload files.
  3. Arrastra tus archivos o carpetas a la zona marcada.
  4. Abajo escribe un commit message, por ejemplo: "Agrego primera versión del proyecto".
  5. Pulsa Commit changes.

Editar archivos directamente en la web

  1. Abre cualquier archivo en tu repo.
  2. Pulsa el icono del lápiz ✏️ arriba a la derecha.
  3. Edita el contenido.
  4. Abajo escribe un commit message y dale Commit changes.

Crear una carpeta nueva desde la web (truco)

Sí, en GitHub puedes crear carpetas, pero hay un truco: cuando pulsas Add file → Create new file, en el campo del nombre escribe:

mi-carpeta/archivo.md

El / es lo que convierte mi-carpeta en carpeta. Luego en VS Code haz git pull para que aparezca localmente.

Conectar tu carpeta local con el repo recién creado

Abre tu carpeta en VS Code, abre la terminal (Ctrl+`) y ejecuta:

# Inicializar Git en la carpeta
git init

# Conectar al repo remoto (pega la URL que copiaste)
git remote add origin https://github.com/tu-usuario/tu-repo.git

# Bajar el README/gitignore que se creó en GitHub
git pull origin main

# Crear archivos, agregar, commit y push
git add .
git commit -m "primer commit local"
git push --set-upstream origin main

Cuando aparece error en el primer push

Si te sale error: failed to push some refs, suele ser porque el remoto tiene commits que tú no tienes localmente (el README). Solución: git pull origin main --allow-unrelated-histories y vuelve a hacer git push.

Botones importantes en la página del repo

Qué hace cada botón en la interfaz de GitHub
BotónPara qué sirve
CodeTe da la URL para clonar el repo o descargarlo en ZIP.
IssuesLista de bugs, tareas o ideas pendientes.
Pull requestsSolicitudes de cambios pendientes de revisión.
ActionsAutomatizaciones (tests, despliegues, CI/CD).
ProjectsTableros tipo Trello para organizar tareas.
WikiDocumentación extendida del proyecto.
SettingsConfiguración (visibilidad, colaboradores, etc.).
Star ⭐Marcar el repo como favorito.
ForkHacer una copia del repo en tu cuenta.
WatchRecibir notificaciones cuando algo cambie en el repo.
